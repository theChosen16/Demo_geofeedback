# Security Policy

## Supported Versions

Currently, the following versions of Geofeedback are supported with security updates:

| Version  | Supported          |
| -------- | ------------------ |
| v1.0.0   | :white_check_mark: |
| < v1.0.0 | :x:                |

## Reporting a Vulnerability

We take the security of Geofeedback seriously. If you discover a security vulnerability within this project, please follow these steps to report it:

1.  **Do not** open a public issue.
2.  Send an email to the maintainers at [alejandro.hernandez.aguirre.16@gmail.com](mailto:alejandro.hernandez.aguirre.16@gmail.com) with a description of the vulnerability.
3.  Provide details such as:
    - Description of the vulnerability.
    - Steps to reproduce.
    - Potential impact.
4.  Please allow up to 48 hours for a response.

We will acknowledge your report and work to address the issue as quickly as possible. We request that you adhere to responsible disclosure practices and give us enough time to resolve the issue before making any information public.

## Disclosure Policy

When a vulnerability is reported, we will:

1.  Acknowledge the report.
2.  Investigate and confirm the vulnerability.
3.  Develop and test a fix.
4.  Release a new version containing the fix.
5.  Provide credit to the researcher (if desired).

Thank you for helping keep Geofeedback secure!
